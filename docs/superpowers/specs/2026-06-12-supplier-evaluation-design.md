# 공급업체 평가 관리 시스템 설계 문서

**작성일**: 2026-06-12  
**대상**: 와이어로프 제조업체 구매팀  
**형태**: Python Flask 로컬 서버 + 웹 프론트엔드 + 자동화 스케줄러

---

## 1. 개요

공급업체 재평가 보고서를 디지털화하고, 평가 이력 관리·자동 평가 생성·이메일 발송·출력/내보내기를 지원하는 시스템.  
담당자는 주기적으로 실적 데이터(납기율/부적합율)만 업데이트하면, 이후 평가·보고서 생성·이메일 발송은 완전 자동 처리된다.

---

## 2. 아키텍처

```
[Windows Task Scheduler] ─ 매일 08:00
        ↓
  scheduler.py
        ├─ D-90/60/30: 경고 이메일 발송
        └─ 평가일 도래 시: 자동 평가 생성 → 보고서 생성 → 이메일 발송 → 기록 저장

[Web App] http://localhost:5000 (Flask)
  ├─ 대시보드
  ├─ 업체관리
  ├─ 실적데이터 입력
  ├─ 평가이력
  └─ 보고서 출력

[Data Layer]
  data/suppliers.json      — 업체 마스터
  data/evaluations.json    — 평가 이력
  data/performance.json    — 납기율/부적합율 실적 데이터
```

## 5. 점수 계산 로직 (scorer.py)

### ISO 9001 (50점)
- 보유 + 유효기간 내: **50점**
- 미보유 또는 만료: **0점**

### 납기율 (25점)
```python
score = min((actual / target) * 25, 25)
```

### 부적합율 (25점)
```python
if target == 0:
    score = 25 if actual == 0 else 0
else:
    score = max((1 - actual / target) * 25, 0)
```

### 등급 및 재평가 주기
| 등급 | 총점 | 결과 | 재평가 주기 |
|------|------|------|------------|
| A | 90 ~ 100 | 승인 | 3년 |
| B | 70 ~ 89  | 승인 | 2년 |
| C | 50 ~ 69  | 조건부 승인 | 1년 |
| D | 0 ~ 49   | 승인불가 | 즉시 재평가 |

## 6. 자동화 흐름 (scheduler.py)

```
매일 08:00 실행
    ↓
모든 업체 nextEvalDate 조회
    ↓
D-90: 1차 경고 이메일 (재평가 예정 안내)
D-60: 2차 경고 이메일 (준비 요청)
D-30: 3차 긴급 이메일 (즉시 대응 요청)
D-0 이하: 자동 평가 실행
    ↓
performance.json에서 최신 실적 데이터 조회
    ↓
scorer.py로 점수/등급/차기평가일 계산
    ↓
report.html 기반 보고서 HTML 생성
    ↓
Gmail SMTP로 이메일 발송 (보고서 HTML 첨부)
    ↓
evaluations.json에 기록 저장 (isAutoGenerated: true)
```

## 7. 웹앱 화면 구성 (4개 뷰)

### 대시보드
- 요약 카드: 전체 업체 수 / 승인 업체 수 / 재평가 임박(D-30 이내)
- 재평가 예정 목록 (D-30: 빨강, D-60: 주황, D-90: 노랑)
- 등급별 현황 (A/B/C/D 개수)

### 업체관리
- 업체 목록 테이블 (검색, 카테고리 필터)
- 업체 등록·수정·삭제 모달

### 실적데이터 입력
- 업체 선택 → 직전 데이터 자동불러오기
- ISO 보유 여부/유효기간, 납기율/부적합율 목표·실적 입력
- Risk 항목 (Critical Supplier만 활성화)
- 결재자명 입력 (직전 데이터 자동불러오기)
- 저장 즉시 점수/등급 미리보기

### 평가이력
- 업체별 전체 평가 타임라인
- 자동생성/수동구분 표시
- 보고서 미리보기, PDF 출력, 수정, 삭제
