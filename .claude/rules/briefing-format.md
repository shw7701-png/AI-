# 브리핑 HTML 형식 규칙

`briefings/` 폴더 작업 시 아래 형식을 준수한다.

## 수신자 프로필

**와이어로프 업체 구매 담당자** — 구매·조달·공급망·제조 분야 AI 동향을 우선으로 본다.

## 뉴스 정렬 우선순위

1. 🔴 핵심 — 구매·조달·공급망·원자재·제조 AI 직접 관련
2. 🟡 관련 — 산업 자동화·B2B 도구·무역 정책 간접 관련
3. ⚪ 참고 — 일반 LLM/AI 동향

## 필수 구성 요소

- 상단: 날짜 + "AI 모닝 브리핑" + "구매 담당자 Edition" 부제
- "LLM & 생성AI 특화" 태그 + "와이어로프·조달" 태그
- 각 뉴스 카드에 업무 연관성 배지 + 한 줄 이유
- **구매 담당자 Action Point 섹션** (실무 적용 인사이트 2~3줄)
- 주목 트렌드 칩 (3~4개)
- 주목 도구·서비스 카드 (연관성 배지 포함)
- 추천 읽을거리 2편
- 하단: "다음 브리핑 — 내일 오전 7:00 KST"
- `sendPrompt` 버튼 2~3개 (구매 업무 관련 심화 질문)

## 스타일 요구사항

- 다크모드 호환 CSS 변수 사용 (`var(--color-*)`)
- 🔴 핵심 카드는 red border-left, 🟡 관련은 yellow, ⚪ 참고는 gray
- 언어: 한국어 요약 + 원문 출처 링크
- 별도 외부 CSS 없이 인라인 스타일 또는 `<style>` 태그 사용

## 파일 명명 규칙

- 저장 경로: `briefings/YYYY-MM-DD.html`
- 오류 발생 시: `briefings/YYYY-MM-DD-error.txt` (커밋 제외)
